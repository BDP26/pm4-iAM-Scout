# Definition of Done für iAM-Scout

Das Projekt *iAM-Scout* gilt als „Done“, wenn folgende Kriterien erfüllt sind:

1. Die Datenpipeline funktioniert stabil und verhindert Duplikate.

2. Die Datenbank ist strukturiert aufgebaut, technisch sinnvoll gewählt und reproduzierbar aufsetzbar.

3. Der Code ist vollständig im Git-Repository versioniert, modular implementiert und wartbar strukturiert.

4. Kritische Kernfunktionen sind durch Unit Tests abgesichert und alle bekannten Fehler behoben.

5. Jede implementierte Funktionalität wurde vom zweiten Teammitglied überprüft.

6. Es existiert ein kleines Frontend, mit dem Nutzer Queries gegen die Datenbank ausführen und Ergebnisse einsehen können.

7. Alle definierten User Stories sowie deren Akzeptanzkriterien sind erfüllt und vom Team abgenommen.
